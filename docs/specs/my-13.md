# MY-13: Search Quality v1 Implementation Spec

## Goal

Improve search quality by adding hybrid dense+sparse retrieval for document chunks, indexing FAQ
entries into a separate Qdrant collection, and adding optional LLM query rephrase before query
vectorization. From the user's perspective, document search should find both semantic matches and
exact corporate terms more reliably, FAQ answers should be searchable separately from documents,
and AI-agent searches should use a cleaner rephrased query without making normal search depend on
LLM latency by default.

        server_default=func.now(),

## API Surface

Existing document search remains authenticated with `get_current_user`.

```python
class SearchFilters(BaseModel):
    file_type: str | None = None
    folder_path: str | None = None
    org_id: int | None = None
    date_from: datetime | None = None
    date_to: datetime | None = None


class SearchRequest(BaseModel):
    query: str
    limit: int = 10
    filters: SearchFilters = SearchFilters()
    use_rephrase: bool = False


class SearchMetadata(BaseModel):
    original_query: str
    effective_query: str
    rephrase_applied: bool
    retrieval_mode: Literal["dense", "hybrid"]


class SearchResponse(BaseModel):
    results: list[SearchResult]
    total: int
    query_time_ms: int
    metadata: SearchMetadata
```

`POST /api/search/`

- Auth dependency: `current_user: User = Depends(get_current_user)`
- Access level: document `viewer` or higher via existing Qdrant payload filter and PostgreSQL permission enrichment
- Request: `SearchRequest`
- Response: `SearchResponse`
- Status codes: `200`, `401`, `422`
- Behavior: document-only hybrid search in `documents`; no FAQ rows in response

FAQ search:

class FAQSearchRequest(BaseModel):

`POST /api/search/faq`

- Auth dependency: `current_user: User = Depends(get_current_user)`
- Required access level: authenticated user; only `FAQ.is_active == True` rows are searchable for non-admin users
- Request: `FAQSearchRequest`
- Response: `FAQSearchResponse`
- Status codes: `200`, `401`, `422`
- Behavior: FAQ-only hybrid search in `faq`; no document rows in response

## Worker Pipeline Impact

B2 changes document indexing:

- Ensure `documents` collection exists with named `dense` and `sparse` vector configs.
- For each chunk, compute dense vector with the worker singleton embedder.
- For each chunk, compute sparse BM25 vector with FastEmbed using `settings.sparse_embedding_model`.
- Upsert one point per chunk with `vector={"dense": dense_vector, "sparse": sparse_vector}`.
- Keep existing payload fields: `document_id`, `title`, `chunk_index`, `text`, `file_type`, `folder_path`, `uploaded_by`, `org_id`.
- Add payload field `indexed_model` with the dense model name and `sparse_model` with the sparse model name for diagnostics.

Reindex strategy:

- Qdrant dense-only `documents` collection cannot serve named-vector hybrid queries.
- Admin reindex must delete/recreate `documents` or create a controlled migration command, then enqueue existing `process_document(document_id)` tasks.
- Existing PostgreSQL document rows remain source-of-truth.

B3 adds FAQ indexing:

- New Celery task `index_faq(faq_id: int) -> None`.
- New Celery task `reindex_faq_collection() -> None`.
- Fetch active FAQ rows from PostgreSQL.
- Index text as `question + "\n\n" + answer`.
- Use stable point id `uuid5(NAMESPACE_DNS, f"faq-{faq_id}")`.
- Delete Qdrant point when `FAQ.is_active` becomes false or FAQ is deleted.
- Update `FAQ.indexed_at` after successful upsert.

B4 adds query rephrase:

- Add an async helper in API code, for example:

```python
async def rephrase_search_query(query: str, *, timeout_seconds: float) -> str:
    ...
```

- Use `httpx.AsyncClient` with settings from `shared/config.py`.
- Prompt must request only a concise search query in Russian or the user's language, without explanations.
- On timeout, provider error, missing API key, or empty response, return the original query and set `rephrase_applied=False`.

## Acceptance Criteria

- `docker compose config -q` succeeds after Qdrant is pinned to `qdrant/qdrant:v1.17.0`.
- `POST /api/search/` returns document results from Qdrant hybrid search and includes `metadata.retrieval_mode == "hybrid"`.
- `POST /api/search/` with `use_rephrase=false` does not call the LLM provider.
- Agent/RAG backend search calls set `use_rephrase=true` and response metadata shows whether rephrase was applied.
- Qdrant `documents` collection has named `dense` and `sparse` vector configs.
- Qdrant `faq` collection exists separately from `documents`.
- `POST /api/search/faq` returns only active FAQ rows and never document rows.
- A non-admin user cannot retrieve inaccessible documents through hybrid search.
- Admin can enqueue FAQ reindex and document reindex.
- Frontend build passes with zero TypeScript errors and no `any`.

## Out of Scope

- Fine-tuning embeddings.
- Gold standard relevance dataset.
- DeepEval or other retrieval evaluation framework.
- Augmented retrieval with generated hypothetical documents or answers.
- Hierarchy-aware indexing by document structure.
- Elasticsearch, OpenSearch, or PostgreSQL full-text search as a second search engine.
- Cross-encoder reranking.
- Changing chunk size or overlap from 2048/256.
- Loading embedding models per request or per task.
- Changing the 50 MB upload cap.
